    ⬡ Design Notes

    The Design Language: Content-First Editorial with Bento Grid

    This design follows the dominant 2025–2026 aesthetic found across high-credibility products — Linear, Vercel, Stripe, and editorial-first SaaS tools. It is called Content-First Editorial with Bento Grid + Glassmorphism accents.

    The philosophy: data-heavy pages need visual hierarchy to remain scannable. A dark background reduces cognitive load by eliminating the high-contrast white canvas that makes densely-typed content fatiguing. The amber accent color (#E8892B) is the dominant non-generic accent color in 2026 product design — it reads warm and trustworthy without the generic tech-blue overuse or the now-played-out purple-to-blue gradient.

    Why it works: The editorial grid creates asymmetric information density — featured content occupies larger cells, secondary content smaller cells. This mirrors how print editorial layouts work: not everything is equal importance, so not everything should be equal size.

    Color System

    #0a0a0a — Near Black
    Primary bg. Near-black (not pure black) prevents harsh contrast that makes text hard to read at night. Pure #000 looks cheap on screens.
    #E8892B — Amber
    2026 dominant accent. Warm, trustworthy, food-appropriate. Orange-amber signals freshness and vitality without the generic blue-tech or overused purple-pink gradient palette.
    #4ade80 — Leaf Green
    Secondary accent. Nature context grounds the design. Green signals freshness, growth, fruit. Used sparingly for in-season indicators and data highlights.
    rgba(255,255,255,0.04) — Glass
    Glassmorphism surface. Used ONLY on the filter panel and overlays — not on every card. Overusing glass is a 2022 anti-pattern. Here it creates surface depth on interactive panels.

    Zero purple appears anywhere on this page. No hex, rgba, hsl, or named purple variant. The blue-purple-pink gradient was the defining overuse of 2023–2024 AI-generated design. Actively avoided here.

    Typography System

    Inter is the choice — it is the highest-quality variable-weight grotesque optimized for screen legibility at small sizes. It pairs the mechanical precision of Helvetica with the warmth needed for long-form reading. The Thai text uses system fallback stack (Arial, sans-serif) as it doesn't interfere with Inter's weight rendering.

    • Hero Thai: clamp(56px, 12vw, 120px) — oversized to signal cultural depth before the English headline
    • Section labels: 11px, uppercase, letter-spacing 0.15em — the label communicates before the content does
    • Card title: 18–22px, weight 600 — readable at small card sizes
    • Body: 14px, line-height 1.65 — looser line-height for dark-background comfort

    Bento Grid: Advantages vs Disadvantages

    Advantages for this use case:

    • Creates visual hierarchy without explicit headers — featured fruits naturally draw attention through cell size
    • Allows progressive disclosure: more info is revealed in the larger cells without overloading small ones
    • Asymmetric density mirrors editorial print design — the brain reads varied-size grids faster than uniform grids
    • Scales gracefully from 4 to 25 items without the "dead end" appearance of equal-height grid + unused cells

    Disadvantages for this use case:

    • Harder to implement consistently responsive — desktop bento requires separate mobile layout
    • Content importance changes by season — a statically-featured cell may not always contain the most relevant fruit
    • Filter reordering can disrupt the bento layout's visual coherence (we reset to standard grid when filtering)
    • Screen readers prefer predictable grid order — requires care with aria attributes

    Glassmorphism: How It's Used (and Why Sparingly)

    Glassmorphism is applied to three surfaces only: the filter sidebar, flavor tag pills, and the detail panel backdrop. This is intentional restraint.

    Overusing glassmorphism is a 2022–2023 anti-pattern. When every surface is frosted glass, none of them have depth — they all look equally floaty. Glassmorphism earns its effect by contrast with solid surfaces. The filter sidebar uses glass to signal it is a layer above the grid, not part of it. The detail panel uses it to communicate overlay state. That's the full extent.

    The 9 AI-Design Tropes: Actively Avoided Here

    These are the most identifiable patterns from AI-generated design tools (2023–2025). Each is explicitly absent from this page.

    1. Hero gradient blobs / soft purple-blue aurora backgrounds Avoided

      Background is #0a0a0a. No radial gradients, no aurora, no color blob. The hero uses typography weight and scale — not visual noise — to create impact.

    2. "Everything is a rounded white card on gray background" Avoided

      Cards use #111111 on #0a0a0a. No white. No generic gray. Border radius is 12px — present but not the rounded-2xl of every generic dashboard.

    3. Emoji bullet points in body text Avoided

      No emojis appear in body copy, lists, or descriptions. The one emoji-like element (⬡ Design Notes button) is a Unicode hexagon used as an abstract mark, not a semantic emoji.

    4. Hollow outlined icons with no weight variation Avoided

      Icons are used minimally — only the search field and close button. Both use consistent 2px stroke weight inline SVG. No icon library, no filler icons.

    5. Blue/purple/pink generic tech gradient palette Avoided

      Zero purple. Zero blue-to-purple gradient. The entire palette is amber (#E8892B), green (#4ade80), and near-black neutrals. Deliberately non-tech-generic.

    6. Centered everything — no real information hierarchy Avoided

      The layout is left-anchored with a fixed filter column driving the reading flow. Cards are left-aligned text. Only the hero headline centers — appropriate for hero sections with no competing content.

    7. CTA buttons that say "Get Started" or "Learn More" Avoided

      There are no generic CTAs. Interactive elements describe their function: "Clear all", "Close", the filter pills all say exactly what they filter.

    8. Fake testimonial cards with generic avatar circles Avoided

      This page contains zero social proof theater. No avatars, no star ratings, no "thousands of travelers love this" copy. Data credibility comes from the information itself.

    9. Section headers with "FEATURES" above big bold text Avoided

      Section labels on this page (like "THAILAND" in the eyebrow) are contextual metadata, not decorative feature-section markers. The detail panel sections use direct functional headings: "How to Eat", "Where to Buy" — not "EATING EXPERIENCE" above "How to Enjoy This Fruit".

    Mobile Responsiveness Approach

    Desktop-primary design with responsive adaptation:

    • Filter sidebar collapses to a slide-up sheet on mobile (triggered by a bottom bar button)
    • Bento grid collapses to single-column card stack on mobile — featured card drops its 2x size advantage
    • Hero typographic sizes use clamp() throughout — no breakpoint-based font-size switching
    • Touch targets on pills and cards meet 44px minimum height
    • Detail panel takes 100% width on mobile vs. 640px max-width on desktop

    Technical Decisions

    • Vanilla JS + no framework — This is a content-display site. A React bundle would add 40KB+ for functionality achievable in 8KB of plain JS. Simpler code, faster load, easier to understand.
    • Pure SVG radar chart — No Chart.js, D3, or Recharts. The flavor radar is generated by trigonometry directly in SVG. Eliminates a 300KB+ dependency for one chart.
    • IntersectionObserver lazy loading — Images load as they enter viewport, not all at once. Critical for a 25-fruit image-heavy page.
    • CSS-only micro-interactions — Card hover states, transitions, and pill active states are pure CSS. No JavaScript event listeners for visual feedback.
    • URL parameter sync — Active filters are reflected in the URL so filtered views are shareable and bookmarkable.
    • WebP images — All images processed to WebP with max 1MB constraint. 30–50% smaller than JPEG at equivalent quality.
